## SECTION 3: GODOT 4 SPECIFIC RESOURCES

### 3.1 Vehicle Physics Systems

#### Godot Easy Vehicle Physics (RECOMMENDED -- Best for arcade/simcade)
- **Godot Asset Library:** https://godotengine.org/asset-library/asset/2558
- **GitHub:** https://github.com/DAShoe1/Godot-Easy-Vehicle-Physics
- **What it is:** Raycast-based rigid body vehicle for Godot 4.2+
- **Key Features:**
  - Works with both Godot and Jolt physics engines
  - 4 example vehicles: arcade, simcade, monster truck, drift
  - All parameters in single script (vehicle.gd) with tooltips
  - Automatic spring/damping calculation
  - Keyboard-optimized controls
- **License:** MIT
- **Best for:** Arcade-style GTA driving feel

#### VitaVehicle (Beta) -- Realistic Car Physics
- **URL:** https://jreo.itch.io/rcp4
- **GitHub:** https://github.com/jreo03/g-rcp2
- **What it is:** Raycast-based car simulator with engine/transmission/slip simulation
- **Key Features:**
  - Realistic engine and transmission simulation
  - Torque graph editor (view in-editor)
  - Powersliding physics
  - Various steering assist modes
  - VitaVehicle 2.0 in development with multi-raycast
- **License:** MIT
- **Best for:** More realistic driving simulation
- **Caveat:** Not very stable with Bullet physics; works better with Jolt.

#### Godot Advanced Vehicle
- **GitHub:** https://github.com/Dechode/Godot-Advanced-Vehicle
- **What it is:** Custom RigidBody car controller with raycast suspension
- **More realistic than built-in VehicleBody3D**

#### Godot 4: Physic Car (Production Ready)
- **URL:** https://joblab-studio.itch.io/godot-4-car-asset
- **What it is:** Pre-configured VehicleBody3D scene with tuned physics and dynamic camera

#### Godot Built-in VehicleBody3D
- **Documentation:** https://docs.godotengine.org/en/stable/classes/class_vehiclebody3d.html
- **What it is:** Built-in vehicle physics node
- **Setup:** VehicleBody3D + CollisionShape3D + 4x VehicleWheel3D children
- **Simplest option** but less configurable than plugins above

## SECTION 5: FILE FORMAT COMPATIBILITY WITH GODOT 4

| Format | Godot 4 Support | Notes |
|--------|-----------------|-------|
| **glTF / GLB** | Native, preferred | Best format. Direct import. Kenney distributes as GLB. |
| **FBX** | Supported (4.0+) | Works well. Most itch.io packs include this. |
| **OBJ + MTL** | Supported | Basic format. No animations. Good for static models. |
| **Blender (.blend)** | Supported (4.0+) | Direct import if Blender is installed. |
| **COLLADA (.dae)** | Supported | Older format, still works. |
| **WAV** | Native | Best for short sound effects. |
| **OGG Vorbis** | Native | Best for music and longer ambient loops. |
| **MP3** | Supported (4.0+) | Works but OGG preferred for looping. |

**RECOMMENDATION:** Prefer glTF/GLB for 3D models and OGG for audio in Godot 4.

## SECTION 6: STYLE CONSISTENCY GUIDE

To maintain visual coherence across all assets:

1. **Primary style:** Low-poly with flat/gradient shading (Kenney aesthetic)
2. **Color palette:** Use Kenney's existing color schemes as reference
3. **Poly count:** Keep individual models under 5,000 triangles
4. **Texture approach:** Single atlas/gradient textures (KayKit style)
5. **Scale:** Standardize all assets to the same unit scale on import

**Style-compatible asset families (these all work together):**
- All Kenney City Kit packs (designed as a series)
- KayKit packs by Kay Lousberg (same designer ecosystem)
- RGS_Dev vehicles (matches low-poly aesthetic)
- Quaternius packs (similar low-poly approach)

**Assets that may need adjustment:**
- Sketchfab individual models (varying styles -- may need material tweaking)
- ArtStation packs (check poly density matches your other assets)
- Poly.pizza models (individually varied -- curate carefully)
